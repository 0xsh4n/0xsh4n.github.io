Manual Web Pentesting Checklist

A comprehensive guide for manual web application security testing

Critical Severity

SQL Injection (Manual)

Test input fields for SQL injection vulnerabilities using manual payloads

  • Test single quotes (') and double quotes (")
  • Try boolean conditions (AND 1=1, AND 1=2)
  • Attempt time-based payloads (SLEEP, WAITFOR DELAY)
  • Test for UNION-based injection

Remote Code Execution

Check for possible command injection points

  • Test file upload functionality
  • Check input fields that might process server commands
  • Verify file inclusion parameters

Authentication Bypass

Test authentication mechanisms for bypass possibilities

  • Try parameter manipulation
  • Test direct page access
  • Check for logic flaws in authentication flow

High Severity

Cross-Site Scripting (XSS)

Test all input fields for XSS vulnerabilities

  • Test basic payloads (<script>alert(1)</script>)
  • Try event handlers (onmouseover, onclick)
  • Check for DOM-based XSS
  • Test HTML injection points

CSRF Vulnerabilities

Check for Cross-Site Request Forgery protections

  • Verify CSRF token implementation
  • Test token validation
  • Check token presence in sensitive operations

Medium Severity

Security Misconfigurations

Check for common security misconfigurations

  • Review HTTP security headers
  • Check for sensitive information in source code
  • Verify error handling messages

Password Policy

Test password requirements and restrictions

  • Check minimum password length
  • Test password complexity requirements
  • Verify password change functionality

Low Severity

Information Disclosure

Look for unnecessary information disclosure

  • Check for version information
  • Review HTML comments
  • Inspect response headers

Cookie Attributes

Verify cookie security settings

  • Check Secure flag
  • Verify HttpOnly attribute
  • Test SameSite attribute
